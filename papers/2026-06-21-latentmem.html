LatentMem: Customizing Latent Memory for Multi-Agent Systems
— Learning a role-tailored latent memory for every agent — 에이전트마다 '역할 맞춤' 잠재 메모리를 학습한다

👥 Muxin Fu et al.외 7인 🏛 Tongji Univ · Shanghai AI Lab · CUHK 외 🗓 2026.02 📄 arXiv:2602.03036
⚡ TL;DR

It tackles the two chronic ailments of multi-agent (MAS) memory at once — role-blind, one-size-fits-all memory (homogenization) and exploding accumulated context (information overload). After retrieving from an experience bank that stacks raw trajectories without any processing, a memory composer compresses them — together with each agent profile — into a fixed-length, role-specific latent memory that is injected as tokens into each agent's reasoning. Trained with LMPO, which backpropagates the task reward through the latent memory — up to 16.20% over vanilla on AutoGen·TriviaQA, an average +6.66%p over multi-agent memory baselines, and 2.16× faster inference.

멀티에이전트 메모리(MAS memory)의 두 고질병 — 역할 구분 없는 '천편일률' 메모리(homogenization)와 누적 컨텍스트 폭증(information overload) — 을 동시에 푼다. 원본 trajectory를 가공 없이 쌓는 experience bank에서 검색한 뒤, agent profile과 함께 memory composer가 역할별 고정 길이 잠재 메모리로 압축해 각 에이전트의 추론에 토큰으로 주입한다. 태스크 보상을 잠재 메모리로 역전파하는 LMPO로 학습 — AutoGen·TriviaQA에서 vanilla 대비 최대 16.20% 향상, 멀티에이전트 메모리 기준선 평균 +6.66%p, 추론 시간은 2.16배 단축.

Why it matters왜 중요한가

Multi-agent memory collapses the moment you hand "the same memory to everyone" — roles break down.

멀티에이전트 메모리는 "모두에게 같은 기억"을 주는 순간 역할이 무너진다.

Existing MAS memory is a one-size-fits-all design that pours the same memory into every agent, so role adherence (role adherence) weakens and correlated errors (correlated error) get amplified — this is memory homogenization. At the same time, multi-granularity memory makes the number of stored items explode, obscuring the very decision signals that matter (information overload). Instead of handcrafted memory, LatentMem targets both at once with a learnable memory compressed per role in the latent space.

기존 MAS 메모리는 모든 에이전트에게 같은 기억을 붓는 one-size-fits-all 설계라, 역할 준수(role adherence)가 약해지고 상관된 오류(correlated error)가 증폭된다 — 이것이 memory homogenization이다. 동시에 multi-granularity 메모리는 저장 항목 수가 폭증해 정작 중요한 결정 신호를 가린다(information overload). LatentMem은 손으로 깎은(handcrafted) 메모리 대신, 잠재 공간에서 역할별로 압축한 학습 가능한 메모리로 이 둘을 한 번에 겨냥한다.

How it works아이디어: 두 개의 축

① 역할 인식

Memory Composer

It takes the retrieved raw trajectory T_q and the active agent's role profile γ as input and produces a fixed-length latent memory m_j ∈ R^(L′×D). Because it is injected by concatenating onto the prompt's hidden state h_j, there is no modification to the agent or system architecture at all. In t-SNE it clusters distinctly by role.

검색된 원본 trajectory T_q와 활성 에이전트의 역할 프로필 γ를 입력받아 고정 길이 잠재 메모리 m_j ∈ R^(L′×D)를 생성. 프롬프트 hidden state h_j에 concat해 주입하므로 에이전트·시스템 구조 수정이 전혀 없다. t-SNE에서 역할별로 뚜렷이 클러스터링된다.

② 학습

LMPO (Latent Memory Policy Optimization)

A variant of GRPO. By treating the latent memory m_j as a differentiable interface, it backpropagates the gradient of the task reward through the agent's forward pass into the composer φ. Being a token-level surrogate rather than trajectory-level, it enables balanced training even in long MAS interactions.

GRPO의 변형. 잠재 메모리 m_j를 미분 가능한 인터페이스로 두어, 태스크 보상의 그래디언트를 에이전트 forward를 거쳐 composer φ로 역전파한다. trajectory-level이 아닌 token-level surrogate라 긴 MAS 상호작용에서도 균형 잡힌 학습이 가능.

  2. Compose합성The memory composer takes T_q plus each agent's role profile and produces a role-specific fixed-length latent memory m_j.memory composer가 T_q + 각 에이전트의 역할 프로필을 받아 역할별 고정 길이 잠재 메모리 m_j를 생성한다.
  3. Inject주입Concatenate m_j(L′×D) onto the prompt encoding h_j(L×D) and reason over the (L+L′)×D expanded input — the backbone stays frozen.프롬프트 인코딩 h_j(L×D)에 m_j(L′×D)를 concat해 (L+L′)×D 확장 입력으로 추론한다 — 백본은 동결.

What to take away그래서, 가져갈 것

🚀

Compression = efficiency + performance. Far from using more tokens, it uses 0.01M fewer and still gains +8.40% — JoyAgent managed only +2.50% with +1.87M tokens. Information overload is resolved by latent compression.

압축 = 효율 + 성능. 토큰을 더 쓰기는커녕 0.01M 적게 쓰고도 +8.40% — JoyAgent은 +1.87M 토큰으로 +2.50%에 그쳤다. 정보 과부하를 잠재 압축으로 해소.

🧠

Per-role memory is the key. Removing the agent profile drops MacNet·KodCode by -6.45%p. "The same memory for everyone" hurts more the more complex the MAS.

역할별 기억이 핵심. agent profile을 제거하면 MacNet·KodCode에서 -6.45%p. "모두 같은 기억"은 복잡한 MAS일수록 더 손해다.

🔬

Robust to retrieval count K. G-Memory degrades from overload at K>3, but LatentMem keeps improving as K grows — extracting only what's useful even from redundant trajectories.

검색량 K에 강건. G-Memory는 K>3에서 과부하로 성능이 떨어지지만, LatentMem은 K가 늘수록 꾸준히 향상 — 중복 trajectory에서도 쓸모만 추출.

🧱

Better than fine-tuning. Under the same compute budget it beats direct backbone fine-tuning (MARTI) in every setting (TriviaQA·AutoGen +11.73%p) — learning only the memory while keeping the backbone frozen.

파인튜닝보다 낫다. 같은 연산 예산에서 백본 직접 파인튜닝(MARTI)을 모든 설정에서 능가(TriviaQA·AutoGen +11.73%p) — 백본은 동결한 채 메모리만 학습.

⚠️ One critical note: The evaluation concentrates on small/mid-sized backbones such as Qwen3-4B·Llama-3.1-8B and GRPO-family training. Because the latent memory is opaque tokens that humans cannot read, interpretability and debugging are hard, and how retrieval/storage cost and the risk of accumulating contaminated trajectories are managed as the experience bank grows remains an open question. Whether "per-role clustering" actually leads to collaboration quality is a causal claim that needs verification beyond the t-SNE observation. ⚠️ 한 줄 비판적 시선: 평가가 Qwen3-4B·Llama-3.1-8B 등 소·중형 백본과 GRPO 계열 학습에 집중돼 있다. 잠재 메모리가 사람이 읽을 수 없는 불투명 토큰이라 해석성·디버깅이 어렵고, experience bank가 커질수록 검색·저장 비용과 오염된 trajectory 누적 위험이 어떻게 관리되는지는 열린 문제. "역할별 클러스터링"이 실제 협업 품질로 이어지는 인과는 t-SNE 관찰을 넘어선 추가 검증이 필요하다.
